    背包（KnapSack）DP#

    这一章节记录背包dp以及各种变种类型的解法和技巧

    问题描述#

    有 n 种物品和一个容量为 W 的背包，第 i 个物品的容量和价值为 w_i,v_i，要求选取的容量总数不大于背包容量且价值总数最大。根据选取第 i 个物品的次数以及各种约束，背包问题会被细分为 0-1 背包、完全背包等。

    0-1背包#

    每种物品只能选取一次或者不选取。

    DP 状态：f_{i,j} 表示只选取前 i 个物品，容量为 j 的背包，最多能够到达的价值

    转移方程：

    逻辑：如果我们当前完成了前 i-1 个物品的选取情况，那么开始考虑第 i 个。第 i 个物品有两种方式，要么选取，要么不选取。

    对于不选取的情况， f_{i,j}=f_{i-1,j}

    对于选取第 i 个物品的情况，f_{i,j}=f_{i-1,j-w_i}+v_i

    所以，对于两种情况，我们就要取最大，于是状态转移方程为：

    边界条件：

    0-1背包问题，不需要特别设定边界，初始化时，将所有 f_j 设为 0 或者负数即可

    空间优化：

    我们采取了二维数组来存储状态 f_{i,j}，但是实际上在更新的时候，我们的 f_{i,j} 只和 f_{i-1,?} 有关，也就是说，我们可以维持某个顺序更新的时候维护一个一维数组即可。 模版代码：

    for(int i = 0; i < n; i++)
    for(int j = W; j = w[i]; j--)
            f[j] = max(f[j], f[j - w[i]] + v[i]);

    注意这里，对于背包容量的循环方向是从高容量到低容量。原因是当枚举容量 j 的时候，比方说，我们在 f_{i,2w_i} 的时候，更新使用了第 i 个物品，那么对于 0-1 背包，应该不能使用了，但是这种情况可能会被二次选取，违背了规则。

    完全背包#

    每种物品能够选取无限多次。

    DP状态、转移方程、边界条件与 0-1 方程完全一致，唯一需要注意的是，在更新时，我们需要从低容量到高容量更新，这样每次就相当于无限次数选取第 i 类物品。

    模板代码：

    for(int i = 0; i < n; i++)
    for(int j = 0; j = W-w[i]; j--)
            f[j + w[i]] = max(f[j + w[i]], f[j] + v[i]);
